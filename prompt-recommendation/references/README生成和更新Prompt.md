# 📋 README生成和更新Prompt

## 🎯 核心目标
**文档清晰 > 自动化** - 手动指定文件类型，生成结构清晰、链接有效的README文档

## 🔍 链接验证工具
**自动验证README中的Markdown链接有效性** - 检查链接格式、编码和路径正确性

## 📋 Prompt模板

### 📝 基础生成模式

```
你是一位README生成专家，请根据我的要求生成结构清晰的README文档。

**生成要求**：
1. **文档结构清晰**：使用清晰的标题层级（H1-H4），每个标题添加相应图标
2. **内容组织合理**：按功能或主题分类组织内容，提供简要说明
3. **文件引用链接**：为所有提到的文档生成正确的Markdown引用链接
4. **特殊字符处理**：正确处理文件名中的空格、括号等特殊字符

**文件引用链接要求**：
- 链接格式：`[文档标题](相对路径/文件名.md)`
- **强制编码规则**：
  - 空格：` ` → `%20`
  - 井号：`#` → `%23`
  - 括号：`(` → `%28`，`)` → `%29`
  - 冒号：`:` → `%3A`
  - 问号：`?` → `%3F`
  - 感叹号：`!` → `%21`
- **GitHub兼容性**：所有特殊字符必须URL编码，确保在GitHub上可以正常点击
- **路径验证**：确保链接路径正确，读者可以点击进入

**输出格式**：
- 清晰的文档结构
- 每个文档1-2句话的简要描述
- 所有文件都有正确的引用链接
- 统一的描述风格和格式

请为以下文件类型生成README：[指定文件类型或路径]
```

## 🛠️ 验证功能

- **相对路径计算**：验证从README到目标文件的路径正确性

### 🔗 编码验证
- **特殊字符编码**：检查URL编码是否符合GitHub标准
- **常见问题检测**：

  - 空格 → `%20`
  - `(` → `%28`，`)` → `%29`
  - `:` → `%3A`
  - `?` → `%3F`
  - `!` → `%21`

### 🚨 问题识别
- **无效链接**：格式错误或路径不存在
- **编码问题**：特殊字符未正确编码
- **路径错误**：相对路径计算错误
- **GitHub兼容性**：链接在GitHub上无法点击

2. **编码规范**：特殊字符正确URL编码
3. **路径存在**：目标文件真实存在
4. **相对路径**：从README位置计算正确
5. **GitHub兼容**：在GitHub上可以正常点击

### ❌ 常见问题类型

## 🔧 修复建议

### 📋 编码修复
```markdown
# 错误格式
[文档标题](./# 文档标题.md)

# 正确格式
[文档标题](./%23%20文档标题.md)
```

### 🔗 路径修复
```markdown
# 错误路径
[文档标题](../错误路径/文档.md)
```

## 🔧 特殊注意事项

### 🚨 链接编码问题
- **问题**：包含`#`符号的文件夹名称导致链接无法点击
- **解决方案**：将`#`编码为`%23`，确保GitHub兼容性
- **验证方法**：在GitHub上测试所有链接的点击有效性

### 📋 文件名处理
- **空格处理**：`文件名.md` → `文件名%20.md`
- **括号处理**：`文件(名).md` → `文件%28名%29.md`
- **井号处理**：`#文件夹#/文件.md` → `%23文件夹%23/文件.md`
- **冒号处理**：`文件:名.md` → `文件%3A名.md`
- **问号处理**：`文件?名.md` → `文件%3F名.md`
- **感叹号处理**：`文件!名.md` → `文件%21名.md`

### 🔍 路径计算
- **相对路径**：从README文件位置计算到目标文件的路径
- **层级关系**：正确处理多级目录的路径关系
- **链接一致性**：所有文档引用都使用统一的链接格式
