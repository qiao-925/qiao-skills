# 高信号复核清单

## 1. 交付前必查五问

1. 开头是否已经出现最重要的结论或主线。
2. 每一段是否都带来新的理解、判断或行动价值。
3. 是否存在两段以上在重复表达同一个意思。
4. 用户读完后，是否更容易决定下一步。
5. 压缩后是否仍保留关键依据、边界和风险。

如果以上有两项答不上来，说明当前输出仍然偏松。

## 2. 常见低信号症状

- 同一个结论先解释、再重说、再总结，出现三遍以上
- 大量“为了更好地”“从某种意义上”“需要注意的是”这类低信息过渡
- 列了很多点，但彼此区分度很弱
- 句子都正确，却没有一条能推动判断或行动
- 用“全面”当理由，把材料全塞进去

## 3. 复核顺序

建议按这个顺序压缩：

1. 先删重复
2. 再删空话
3. 再合并相近观点
4. 再前置主结论
5. 最后检查边界、风险和假设是否仍在

这个顺序能避免一上来就把正文剪坏。

## 4. 多轮压缩原则

### 第一轮

- 保住主线
- 去掉明显冗余
- 不急着把内容压到最短

### 第二轮

- 合并相近观点
- 重写长句和弱表达
- 强化结论与行动项

### 第三轮

- 只保留真正高信号部分
- 检查是否已经短到影响准确性
- 若继续压缩会失真，应明确停止

## 5. 停止压缩的信号

出现以下任一情况，应停止继续压缩：

- 再删就会丢失关键假设或适用边界
- 再删就会让结论看起来过度确定
- 再删就会让用户不知道为什么这样判断
- 再删就会失去可执行下一步

目标不是“最短”，而是“最有用”。
