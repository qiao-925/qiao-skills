# 精简与重构模式

## 1. 三分处理法：保留 / 删除 / 重构

### 保留

优先保留以下内容：

- 真正改变理解的核心结论
- 支撑结论成立的关键依据
- 会改变结论方向的假设、边界和风险
- 直接指导下一步行动的建议
- 具有明显解释力或独特性的洞察

### 删除

优先删除以下内容：

- 同义重复或换说法重复
- 没有新增信息的铺垫句
- “显得完整”但不改变判断的背景枝节
- 空泛正确话、口号式表达、礼貌性废话
- 已被更强表达覆盖的弱观点

### 重构

以下内容通常不该直接删掉，而应重构：

- 一句话里混了多个层级的信息
- 重要但表述过长、过绕的判断
- 多段都在说同一主题，但没有合并
- 观点本身有价值，但缺少 why it matters
- 建议本身成立，但没有落到动作层

## 2. 默认压缩骨架

### 复杂回答

优先骨架：

1. 结论
2. 关键依据
3. 主要边界或风险
4. 下一步

### 长文总结

优先骨架：

1. 核心观点 `3-5` 个
2. 关键洞察 `2-3` 个
3. 行动建议 `1-2` 个

### 现有草稿优化

优先骨架：

1. 保留什么
2. 删除什么
3. 重构什么

## 3. 常用压缩动作

### 合并同类项

- 把多条相近结论合并成一条更强判断
- 把多段分散的同一主题并成一段
- 把重复出现的例子压到一个最有代表性的例子

### 前置主结论

- 先说“结论是什么”
- 再说“为什么成立”
- 最后说“什么时候不成立”或“接下来怎么做”

### 句式提纯

- 把抽象话改成具体判断
- 把“看起来很专业”的长句改成更直接的表达
- 把多层从句拆成短句，但保留逻辑关系

### 从描述转向动作

- 把“这个问题很重要”改成“为什么重要”
- 把“可以进一步考虑”改成“下一步先做什么”
- 把“内容需要优化”改成“删什么、留什么、怎么改”

## 4. 精简时不能一起删掉的东西

以下内容只要会影响用户判断，就不能因为“追求短”而删掉：

- 关键假设
- 主要风险
- 适用边界
- 强反例
- 结论力度说明

精简的正确做法，是删掉无效体积，而不是删掉判断完整性。
