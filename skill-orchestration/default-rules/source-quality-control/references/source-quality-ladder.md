# 来源等级与结论力度

> 这份文档提供 `P0 / P1 / P2` 来源分级、纳入规则和结论力度校准标准。适用于所有依赖外部知识的任务。

## 1. 来源分级

### P0：一手核心来源

关键结论优先由这一层支撑：

- 官方文档
- 官方博客 / 工程博客
- 官方 release notes / changelog
- 官方 GitHub 仓库、README、设计说明
- 官方 conference talk
- 核心维护者、创始人或发布方的原始说明
- 标准文本、RFC、原始数据、原论文

### P1：高可信补充来源

用于补充解释、交叉验证或补全背景：

- 大厂官方技术架构文章
- 顶级会议 / 权威研究论文
- 核心维护者访谈
- 高可信机构报告
- 高质量综述或技术白皮书

### P2：二级参考来源

只能补充，不能主导关键结论：

- 高质量技术分析文章
- 社区总结
- 第三方测评
- 非原始的经验贴、教程、博客二次解读

## 2. 纳入规则

- 关键结论尽量由 `P0` 支撑。
- 结论较强、较新、解释性较强时，优先交叉验证。
- 若只能拿到 `P2` 证据，必须降低结论力度。
- 若来源之间冲突，优先采用距离原始事实更近、解释权更强、更新更晚的来源。

## 3. 结论力度校准

| 来源情况 | 建议结论力度 |
| --- | --- |
| 多个 `P0` 一致 | 可给较强结论，但仍应说明边界 |
| 单个 `P0` 或 `P0 + P1` | 可给倾向性结论，并说明假设 |
| 只有 `P1` | 给保守建议，尽量补充交叉验证 |
| 只有 `P2` | 只能给弱结论，并明确不确定性 |

## 4. 事实、推断与建议

- 事实：必须可追溯到具体来源。
- 推断：必须显式写明“这是基于资料的推断”。
- 建议：必须说明为什么当前建议优于替代路径。

## 5. 通用检查清单

- 当前结论的主支撑来源属于哪一级。
- 是否存在更高等级来源尚未核对。
- 是否因为来源偏弱而需要收缩语气。
- 是否已经保留出处、时间点和上下文。
