# 句子清晰化模式

## 1. 先放主干，再补细节

优先顺序：

1. 先说主结论或主要动作
2. 再补原因、条件或解释
3. 最后再放例外、提醒或举例

如果一个句子要读到最后才知道重点，读者中途很容易丢线。

## 2. 该拆就拆

以下情况通常应该拆句：

- 一个句子里有两个以上平级判断
- 主句后面挂了过长的原因、条件和例子
- 读者需要反复回看才能找到主语和谓语
- 本来可以分成“结论句 + 解释句”，却硬写成一整句

目标不是句子越短越好，而是读者一次就能抓住主干。

## 3. 破折号的典型用法

破折号适合做“即时补充说明”，常见场景：

- 对前一句做快速翻译或再解释
- 补一个读者此刻最需要知道的限定
- 把抽象判断立刻落到更具体的意思
- 在不另起一句的情况下补一个短提醒

示例：

- 这个规则只在高风险决策里加强执行力要求——低风险小修正不必强行升级。
- 这里的“结构优化”不是多加抽象——而是减少依赖混乱和职责交叉。

使用注意：

- 一句里默认最多一次，避免破折号套破折号
- 破折号前半句本身应尽量能独立成立
- 不要用破折号拖出一个新的大主题

## 4. 冒号的典型用法

冒号适合做“读者预告”：

- 先给判断，再列展开项
- 先给概括，再列解释
- 先给提醒，再列具体风险

示例：

- 这个回答需要保留三件事：结论、依据、下一步。
- 问题出在两个地方：职责没切开，依赖方向也反了。

## 5. 抽象话改成具体话

优先把下面这类表达改具体：

- “需要进一步优化”
- “存在一定问题”
- “提升整体质量”
- “增强用户体验”

更好的写法是直接说：

- 优化什么
- 问题具体体现在哪
- 为什么这件事重要
- 读者接下来该做什么

## 6. 节奏控制

让句子更顺的常见方法：

- 长短句交替，不连续堆长句
- 关键句单独成段，给读者停顿点
- 同类句式适度平行，方便扫描
- 段首先立意，段中再展开

如果每句话都很长、每段都很满，读者会在“看起来都重要”里失去重点。
