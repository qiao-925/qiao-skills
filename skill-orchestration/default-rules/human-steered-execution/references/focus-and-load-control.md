# 聚焦与认知负荷控制

## 1. 单线程聚焦

默认结构：

- `primary thread`：当前唯一主问题
- `parking lot`：相关但暂不展开的问题

如果多个问题同时出现，先做主次划分，再推进，不直接并行展开。

## 2. 链路推进

多步骤任务优先使用这四个锚点：

1. 当前在做什么
2. 为什么先做这个
3. 做到什么算完成
4. 完成后下一步是什么

这四个锚点越清楚，人类越容易跟上执行节奏。

## 3. 停车区机制

遇到相关但非当前关键的问题时：

- 记录它
- 说明它为什么不是当前优先级
- 说明何时再回来处理

停车区的价值在于：

- 不丢问题
- 不抢主线
- 不制造认知拥堵

## 4. 控制输出体量

输出应优先满足当前判断需要，而不是展示 Agent 的联想能力。

出现以下信号时，应主动收缩输出：

- 候选方案过多
- 分支过多
- 术语过多
- 同级重点超过 `3` 个
- 用户显然还没有消化上一层信息

## 5. 常见失真

- 看似系统，实则把多个问题混在一起
- 看似全面，实则没有当前主步骤
- 看似高效，实则把用户重新丢回发散状态
