# 文件级：一个文件围绕一个稳定职责

## 核心问题

文件级职责失控，最常见的表现不是“大”，而是“杂”。

判断一个文件是否健康，先问两件事：

- 能不能一句话说明这个文件做什么
- 文件里的函数、类是否都服务于这句话

## 典型问题模式

### 1. 工具杂烩

一个文件里同时放日期处理、JSON 解析、邮件发送、哈希计算这类不相关能力。

这类文件的表面好处是“方便找”，实际问题是：

- 变化原因完全不同
- 命名会越来越泛
- 测试和维护边界越来越模糊

### 2. 上帝类文件

一个文件里的主类同时做解析、验证、执行、格式化、缓存、日志。

这通常意味着文件职责已经不是“一个能力”，而是“整个流程的杂糅实现”。

## 拆分原则

- 按职责边界拆，不按代码块大小平均切。
- 如果几个函数总是围绕同一稳定意图变化，可以保留在同一文件。
- 如果几个函数只是恰好都叫“工具”，但变化原因不同，应拆开。

## 拆分建议格式

```text
当前问题：
- ...

建议拆分：
- a.py：...
- b.py：...

拆分理由：
- ...
```

## 检查清单

- [ ] 文件职责能一句话说明。
- [ ] 文件内成员都围绕同一职责。
- [ ] 不存在明显不相关的功能堆叠。
- [ ] 拆分后文件名会更准确，而不是更模糊。
