# 闭环产物契约

## 1. 最小输出骨架

任何使用本 skill 的任务，默认至少沉淀以下五项：

### 1.1 当前认知

说明当前最合理的理解版本是什么。

最低要求：

- 问题定义清楚
- 关键背景到位
- 当前结论有证据边界

### 1.2 核心假设

列出当前最值得验证的假设，而不是泛泛罗列所有可能性。

最低要求：

- 1-3 个最关键假设
- 每个假设都能被后续动作触碰
- 明确它和当前问题的关系

### 1.3 下一步最小行动

说明下一步做什么，以及为什么先做它。

最低要求：

- 动作足够小
- 能减少关键不确定性
- 执行成本和反馈周期可接受

### 1.4 反馈观测点

说明执行之后要重点看什么信号。

最低要求：

- 支持当前判断的信号
- 削弱当前判断的信号
- 可能的噪声项

### 1.5 下一轮修正方向

说明如果反馈出来之后，认知和动作将怎么调整。

最低要求：

- 哪些方向继续推进
- 哪些方向收缩或放弃
- 下一轮重点变量是什么

## 2. 推荐表达模板

```text
当前认知：
- ...

核心假设：
- 假设 A：...
- 假设 B：...

下一步最小行动：
- ...

反馈观测点：
- 若出现 ...，说明 ...
- 若出现 ...，说明 ...

下一轮修正方向：
- ...
```

## 3. 文档化场景下的扩展骨架

若任务需要形成正式文档、研究记录或阶段计划，可扩展为：

1. 问题定义
2. 当前认知
3. 证据与约束
4. 核心假设
5. 下一步最小行动
6. 反馈观测点
7. 下一轮修正方向

## 4. 常见反模式

- 没有“下一步最小行动”，只有大方向口号
- 没有“反馈观测点”，导致执行后无法判断学到了什么
- 假设写得太虚，不能被验证
- 行动过大，导致闭环周期失控
- 产物只记录结论，不记录修正过程
