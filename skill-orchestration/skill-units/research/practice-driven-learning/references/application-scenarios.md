# 典型应用场景

## 1. 技术研究

适用情况：

- 对某个新框架、新系统、新平台还没有稳定判断
- 需要先调研，再做小规模验证

推荐联动：

- `source-quality-control`
- `critical-thinking-evaluation`
- `knowledge-synthesis`

## 2. 架构方案验证

适用情况：

- 已经有候选方案，但不确定是否值得落地
- 需要通过 PoC、局部试验或影响面分析验证判断

推荐联动：

- `architecture-governance`
- `roi-value-density`
- `critical-thinking-evaluation`

## 3. 学习路线推进

适用情况：

- 不是单纯看资料，而是希望边学边做
- 需要把“阅读、理解、练习、反馈”接成一个可持续循环

推荐联动：

- `general-understanding-research`
- `knowledge-synthesis`

## 4. 调试与排障

适用情况：

- 问题暂时没有明确根因
- 需要先形成假设，再用最小动作排除变量

推荐联动：

- `critical-thinking-evaluation`
- `source-quality-control`

## 5. 重构或优化探索

适用情况：

- 已察觉系统有问题，但最优改法不明
- 需要通过小改动、局部试验或数据反馈验证方向

推荐联动：

- `core-first-simplicity`
- `single-responsibility`
- `architecture-governance`
- `roi-value-density`

## 6. 使用边界

以下情况不建议让本 skill 主导：

- 已经有稳定方案，只需直接执行
- 只是把现有资料整理成文档
- 用户只需要单次事实回答，不需要闭环推进
