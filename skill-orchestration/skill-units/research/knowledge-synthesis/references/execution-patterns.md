# 执行模式

## 1. 首轮生成

适用于从零开始构建内容时。

重点：

- 先搭主线
- 再补证据
- 最后精修表达

不要一开始就把所有资料都塞进正文。

## 2. 二轮优化

适用于已有初稿或旧文档时。

重点：

- 保留正确内容
- 修复主线不清、重复表达、结论无力
- 强化因果、洞察和阅读路线

不要为了制造“优化感”而全量重写。

## 3. 多 Agent 分工

推荐方式：

- 主控定义边界和目录
- worker 分工采集和整理证据
- 主控统一编排终稿

不推荐方式：

- 多个 worker 各写一篇完整终稿再直接拼接

## 4. 最小必要收敛

“避免过度收敛”是对的，但“完全不收敛”通常会导致：

- 术语混乱
- 主线断裂
- 章节重复
- 结论力度不统一

因此更合理的原则是：

- 只收敛必要边界
- 不微操具体措辞
- 保留 worker 在证据整理和局部表达上的自由
