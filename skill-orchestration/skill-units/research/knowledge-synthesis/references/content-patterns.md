# 内容构建模式

## 1. 历史优先

当对象存在明显演化过程时，优先组织：

- 起点是什么
- 关键转折是什么
- 每次变化解决了什么问题
- 今天的形态是如何被一步步推出来的

重点不是列年份，而是解释前因后果。

## 2. 设计意图优先解释

高价值内容应尽量补出：

- 设计目标
- 约束条件
- 关键取舍
- 为什么这个方案在当时成立

如果没有这些，读者只能看到表面结构，看不到其成立原因。

## 3. 阅读路线梗概

开篇应回答：

- 这篇内容的主线是什么
- 会先讲什么，再讲什么
- 重点结论在哪里

这样读者进入正文时就不会迷失在细节里。

## 4. 速查层

适合下沉为附属结构的内容：

- 结论清单
- 关键概念对照
- 重要时间点速查
- 核心组件速查

速查层是帮助快速回看，不是替代正文推理过程。

## 5. Focus Value

材料越多，不代表认知越强。

真正高价值的内容要做到：

- 重点突出
- 因果清楚
- 结论有证据
- 观点有洞察

宁可减少覆盖面，也不要让主线被材料淹没。
