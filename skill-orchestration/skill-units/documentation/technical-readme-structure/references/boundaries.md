# 适用边界

## 它适合什么

本 skill 适合：

- 开源技术项目 README
- 工程仓库入口 README
- 框架、工具、服务、平台类代码仓库 README
- 需要帮助技术读者快速启动和理解结构的项目

## 它不适合什么

本 skill 不适合：

- 公司介绍页
- 产品营销页
- 业务方案说明
- 个人主页型仓库简介
- 非技术型通用文案页面

## 判断问题

在使用前先问：

1. 读者主要是不是开发者、维护者或技术协作者？
2. 文档首要目标是不是帮助读者启动、理解结构和配置？
3. 仓库里是否真的存在代码、依赖、配置与技术架构可供核对？

如果以上大多是否定的，就不应由本 skill 主导。

## 易混淆场景

### 产品型仓库

如果仓库既有产品介绍，又有明确的技术启动与结构说明需求，README 可偏技术入口，宣传内容应克制，不要反过来主导结构。

### 模板仓库

如果模板仓库的核心价值是“如何快速使用模板”，仍然适合用本 skill，但需要把快速开始和目录说明写得更具体。

### 单脚本仓库

如果只是一个极小脚本仓库，可以保留简化版结构，但至少仍应回答：它做什么、怎么运行、需要什么输入。
