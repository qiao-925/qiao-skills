## 填写提示

- 一句话描述优先写“问题 + 用户 + 价值”，避免空泛口号。
- 快速开始只保留首轮上手必需步骤，部署细节可外链到其他文档。
- 架构图下必须补文字说明，避免只给图不给语义。
- 目录结构说明只写核心目录，避免把整个仓库原样贴满。
- 配置说明要明确敏感项来源，不要把密钥直接写进示例值。
