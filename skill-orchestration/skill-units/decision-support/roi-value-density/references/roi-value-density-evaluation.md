# ROI 与价值密度评估手册

## 1. 一句话原则

- `ROI` 用来判断一个候选是否值得进入当前决策池。
- `价值密度` 用来判断在当前阶段谁更值得先推进。
- 先看值不值得做，再看现在先做哪个，不要把这两个问题混在一起。

## 2. 默认判断流程

### 2.1 先定义决策问题

先把问题说清楚，再开始比较：

- 当前目标是什么。
- 当前阶段是什么。
- 当前最主要的阻塞或不确定性是什么。
- 当前是在做“进入候选池”的判断，还是“候选池内排序”的判断。

### 2.2 再收敛候选

- 默认收敛到 `2-3` 个候选。
- 候选应围绕同一个目标，否则比较结果没有意义。
- 如果候选之间的目标不同，先拆成多个决策问题。

### 2.3 先看 ROI

一个候选通常更值得进入决策池，当它具备以下信号：

- 直接命中当前核心目标，而不是边缘优化。
- 能明显改善用户结果、交付效率或维护质量。
- 回报与投入大体匹配。
- 即使失败，也能留下明确的学习价值或中间产物。

### 2.4 再看价值密度

一个候选通常更适合“现在做”，当它具备以下信号：

- 能直接缓解当前主线阻塞。
- 验证周期短，反馈快。
- 新增复杂度低到中。
- 做完后能显著提升下一步判断或执行效率。

### 2.5 最后给出动作结论

默认收敛到以下四类结论之一：

- `现在做`：ROI 成立，且当前价值密度最高。
- `稍后做`：ROI 成立，但当前阶段不是最优先。
- `暂不做`：ROI 不成立，或投入与回报明显失衡。
- `先补证据`：关键假设未验证，当前不适合硬下结论。

## 3. 核心比较维度

### 3.1 预期价值

- 是否解决当前真正的问题，而不是表面症状。
- 是否提升用户结果、业务结果、交付效率或维护质量。
- 是否能减少返工、等待、沟通摩擦或后续混乱。

### 3.2 总投入

- 需要多少时间、人力、协调成本和后续维护成本。
- 是否依赖较多前置条件。
- 是否需要额外的组织推动或长期承诺。

### 3.3 新增复杂度

- 是否引入新的抽象层、配置层、流程或系统耦合。
- 影响范围是局部可控，还是会扩散到全局。
- 是否会明显提升团队的认知负担。

### 3.4 验证速度

- 多快能知道这个候选是否有效。
- 能否先做小步验证，而不是一上来大投入。
- 一旦发现方向不对，是否容易回退或止损。

### 3.5 不确定性

- 关键假设是否清楚。
- 是否依赖未确认的数据、接口、用户反馈或组织条件。
- 如果这些假设不成立，损失会有多大。

## 4. 五类高频场景

### 4.1 产品取舍

适合问的问题：

- 这个功能是否真的命中当前用户核心问题。
- 它会不会只是“看起来高级”，但验证价值很慢。
- 如果现在不做，代价是什么。

高 ROI 信号：

- 明确改善核心用户路径或关键业务指标。
- 与当前产品阶段强相关。
- 不做会持续损失机会或放大主线问题。

高价值密度信号：

- 可以快速上线小版本验证。
- 对现有系统和团队负担较小。
- 能帮助后续更多产品判断收敛。

### 4.2 技术选型

适合问的问题：

- 方案是否真正适配当前需求，而不是未来想象。
- 引入它的迁移成本和维护成本是否可接受。
- 是否会带来明显锁定或隐性复杂度。

高 ROI 信号：

- 能解决当前真实的性能、稳定性、交付或维护问题。
- 与现有技术栈兼容性较好。
- 后续扩展成本可控。

高价值密度信号：

- 可以先做局部验证或小范围接入。
- 不需要一次性全局迁移。
- 很快就能验证核心收益是否成立。

### 4.3 重构是否值得做

适合问的问题：

- 当前痛点是否频繁出现、是否真实影响交付。
- 重构是解决根因，还是只在满足审美。
- 是否存在更便宜的止血方案。

高 ROI 信号：

- 问题反复出现，已经持续制造返工或故障风险。
- 不处理会继续抬高未来开发成本。
- 重构后有明确的维护收益或边界清晰度收益。

高价值密度信号：

- 可以局部切入，而不是整体系重写。
- 有清晰的验收标准和回归验证路径。
- 能先处理最痛的一段，而不是一口吃完整个系统。

### 4.4 任务先后顺序

适合问的问题：

- 哪个动作会最快解除当前阻塞。
- 哪个动作做完后能带动更多后续动作推进。
- 哪个动作虽然重要，但现在证据不足。

高 ROI 信号：

- 与当前里程碑或交付目标直接相关。
- 能减少等待、返工或反复讨论。
- 做完后对主线推进有明显帮助。

高价值密度信号：

- 可以在短时间内拿到确定反馈。
- 依赖关系简单。
- 完成后能显著提升全局节奏。

### 4.5 学习路线判断

适合问的问题：

- 这条学习路径是否服务于当前目标，而不是纯兴趣漫游。
- 学完后能否很快用起来，形成正反馈。
- 前置门槛是否过高，导致长期无法兑现价值。

高 ROI 信号：

- 与当前项目、岗位或近期任务直接相关。
- 学完后复用频率高。
- 能明显提升当前执行质量或判断能力。

高价值密度信号：

- 学习周期短，反馈快。
- 可以边学边用，而不是长期囤积知识。
- 对当前问题有立刻可验证的帮助。

## 5. 推荐输出骨架

如果需要轻量输出，可直接按下面骨架组织：

```text
[候选名]
- 当前问题：它解决了什么关键问题
- ROI：高 / 中 / 低
- 价值密度：高 / 中 / 低
- 当前建议：现在做 / 稍后做 / 暂不做 / 先补证据
- 为什么：一句话解释比较结果
- 主要风险：一句话说明待验证点
```

如果只有一个候选，也至少回答四件事：

- 它解决什么问题。
- 为什么整体值得做或不值得做。
- 为什么现在做或不现在做。
- 还缺什么证据。

## 6. 常见止损与延后信号

出现以下情况时，应优先考虑 `暂不做` 或 `先补证据`：

- 候选没有清晰命中当前目标。
- 主要回报停留在想象层，没有验证路径。
- 投入和复杂度明显大于当前阶段能承受的范围。
- 关键假设尚未确认，且一旦错判代价较高。
- 当前存在更直接、更低成本、更易验证的替代动作。

## 7. 反模式

- 把“做了很多”误当成“ROI 很高”。
- 把“学了很多”误当成“现在最值得学”。
- 因为候选 A 更宏大，就默认优先于候选 B。
- 用一堆维度打分，却不解释为什么这些维度对当前阶段重要。
- 明明只是偏好判断，却包装成客观必然结论。
