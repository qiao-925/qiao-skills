# Strategic Fit 视角

## 1. 目的

`Strategic Fit` 用来回答：这个方向是否服务当前主线，还是会让项目越做越散。

它适合资源有限、方向容易发散、需要保护核心路径的场景。

## 2. 什么时候用

- 团队手里有很多看起来都不错的方向。
- 用户担心功能越堆越散，偏离核心定位。
- 需要区分"重要"和"符合当前主线"。

## 3. 不适合什么时候用

- 当前主要问题仍然是不知道用户要完成什么任务。
- 当前争议更像流程卡点或高风险假设，而不是主线聚焦。

## 4. 核心问题

1. 当前项目最重要的核心目标是什么？
2. 这个方向服务哪个核心用户或关键结果？
3. 它会强化主线，还是制造旁支？
4. 如果现在做，会不会挤占更关键的投入？

## 5. 最小判断步骤

1. 明确当前阶段的核心目标和主线。
2. 检查候选方向是否直接服务核心用户、核心结果或核心能力。
3. 标记强化主线、弱相关、明显偏离三类方向。
4. 再回到 `ROI + 价值密度` 判断符合主线的方向里哪个更值得先做。

## 6. 推荐输出

```text
主视角：Strategic Fit
- 当前主线：……
- 强化主线的方向：……
- 偏离主线的方向：……
- 当前建议：……
```

## 7. 反模式

- 把所有有价值的点都当成战略重点。
- 没有先定义主线就谈战略匹配。
- 只因为方向"有趣"或"未来可能有用"就提前投入。
- 用 Strategic Fit 替代真实用户问题分析。
