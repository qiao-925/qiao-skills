# Assumption Mapping 视角

## 1. 目的

`Assumption Mapping` 用来回答：当前方向里什么必须为真，哪个关键假设最值得先验证。

它适合高不确定性的新方向、新功能或新市场判断。

## 2. 什么时候用

- 用户在考虑新方向，但投入前景不清。
- 团队争议集中在"它到底行不行"。
- 当前最重要的不是补更多功能，而是减少错误投入。

## 3. 不适合什么时候用

- 问题已经很清晰，只差落地规划。
- 当前候选项争议主要是基础 vs 加分，而不是不确定性。

## 4. 核心问题

1. 这个方向成立，哪些前提必须为真？
2. 哪些假设一旦不成立，方向就不值得继续？
3. 哪些假设既重要又不确定？
4. 能否用更小成本先验证关键假设？

## 5. 最小判断步骤

1. 列出当前方向的关键假设。
2. 从重要性和不确定性两个维度做粗分。
3. 优先找出高重要、高不确定的假设。
4. 提出最小验证动作，而不是直接大投入实现。
5. 回到 `ROI + 价值密度` 判断当前是该验证、推进，还是延后。

## 6. 推荐输出

```text
主视角：Assumption Mapping
- 关键假设：……
- 最大风险：……
- 最小验证动作：……
- 当前建议：先验证 / 可推进 / 暂缓
```

## 7. 反模式

- 没有列出假设，就直接讨论方案实现。
- 把所有不确定点当成同等重要。
- 用完整开发替代最小验证。
- 把验证建议写成详细实施计划。
