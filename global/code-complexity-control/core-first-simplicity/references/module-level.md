# 模块级：边界清晰与依赖最小化

## 核心原则

模块的职责要少而清晰，依赖要短而稳定。

## 模块设计规则

- 一个模块只承担一个稳定的业务意图。
- 对外接口数量保持最小。
- 模块间依赖方向保持单向，避免循环依赖。

## 简化动作

- 合并高度耦合且总是同时变更的小模块。
- 拆分“超大模块”中不相关职责。
- 去掉仅用于“可能复用”的抽象层。

## 与 SRP 的关系

- SRP 解决“职责是否单一”。
- 本原则解决“是否有多余复杂度”。
- 两者联用：先定边界，再做复杂度裁剪。

## 检查清单

- [ ] 模块职责能一句话说明。
- [ ] 模块导出接口是否可再缩减。
- [ ] 是否存在没有明显收益的中间层。
