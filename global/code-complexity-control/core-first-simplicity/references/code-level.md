# 代码级：结构收敛与实现直达

## 核心原则

代码应优先表达业务意图，而不是展示抽象技巧。

## 编码决策规则

- 能直接写清楚，就不要增加额外抽象。
- 能复用已有结构，就不要并行造新模式。
- 能通过命名表达，就不要依赖隐式魔法。

## 简化动作

- 减少不必要的配置层和适配层。
- 减少深层嵌套与过多间接调用。
- 保持主路径可顺序阅读。

## 反模式

- 过早泛型化、模板化。
- 为了“优雅”拆成过多碎片文件。
- 在无性能压力下做复杂优化。

## 检查清单

- [ ] 核心流程是否可在短路径内读完。
- [ ] 新增抽象是否有明确复用或隔离价值。
- [ ] 代码结构是否比改动前更易理解。
