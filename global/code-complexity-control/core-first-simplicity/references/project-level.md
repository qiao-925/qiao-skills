# 项目级：核心价值与范围控制

## 核心原则

项目级复杂度控制的目标是：让资源集中在“最小光辉点”。

## 必问三题

1. 如果用户只能记住一个亮点，这个亮点是什么？
2. 当前需求是否直接增强该亮点？
3. 不做这件事，核心价值是否受损？

## 决策规则

- 直接增强核心亮点：做。
- 间接支持但非关键：做最小版本。
- 与核心无关：延后或砍掉。

## 常见简化动作

- 合并相似功能，减少并行路线。
- 砍掉“展示型”但不影响核心价值的需求。
- 将高成本优化排到核心闭环验证后。

## 检查清单

- [ ] 当前迭代目标可一句话描述。
- [ ] 本次新增功能全部可映射到核心价值。
- [ ] 非核心需求有明确延后列表。
