# Anthropic Enhancements

## 来源

- 来源页面：`https://skills.sh/anthropics/skills/skill-creator`
- 说明：本文件提炼其中可跨平台复用的通用方法，并显式剔除平台绑定内容。

## 吸纳范围（平台无关）

### 1) 写作与规则表达

- 把 skill 写成“可执行规则”，而非概念描述。
- 优先提供可判断、可检查、可落地的条目。
- 使用动作词与祈使句，避免模糊表达。

### 2) 自由度分级

- 高自由度（原则型）：多解任务，依赖上下文判断。
- 中自由度（模式型）：有推荐路径，允许参数化变化。
- 低自由度（约束型）：高风险/高脆弱任务，使用硬约束清单。

### 3) 渐进式披露

- 主文档保留核心步骤，详细内容放 `references/`。
- 引用保持单层：`SKILL.md -> references/*.md`。
- 避免链式引用和重复维护。

### 4) 维护治理

- 维护动作覆盖：`update` / `refactor` / `migrate` / `deprecate`。
- 变更前后均执行一致性检查（命名、description、引用、边界）。
- 弃用时给出替代路径与迁移说明。

### 5) 反模式识别

- 平台绑定命令或私有字段进入核心规则。
- 主文档过长且无分层。
- “何时使用”只写正文，不写入 `description`。
- 变更后不更新引用与迁移信息。

## 剔除范围（不纳入本 skill）

- 任何 Cloud/IDE/产品私有流程与命令。
- 平台专属打包、安装、发布、统计信息。
- 依赖特定平台脚本链路的强绑定要求。

## 与官方 Step 1–4 的融合原则

- 官方 Step 1–4 作为主流程骨架。
- 本文件内容仅作为各步骤的增强项。
- 增强不替换主流程，不改变官方结构。
