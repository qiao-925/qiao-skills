# 批注式阅读与整理工作流

> **核心比喻**：思想的磨刀石——脑子是刀，输入是磨刀石，产生的火花就是批注。  
> **产出形态**：沉淀下来的内容更像二创；一举多得：沉浸式阅读、与既往所见所想所行建立联系、可复用笔记、可检索的联想与辩证。

---

## 📋 工作流总览

```
输入（原文/案例）→ 沉浸式阅读 + 批注（火花）→ 结构化整理（含批注）→ 二创式沉淀
         ↑                                                              ↓
         └──────────────── 良性循环：后续阅读可再勾连此沉淀 ──────────────┘
```

---

## 🎯 批注模式：三步

### 1. 输入

- **素材**：长文、案例、访谈、报告等（如 InfoQ 文章、X 长文、博客）
- **心态**：不追求「读完」，追求「读到有火花处就停笔」

### 2. 批注（产生火花）

在阅读过程中，一旦某段话触发联想、类比、辩证或与自身经历/思考/行动勾连，**立刻写下批注**。

**批注写什么**（保持原话，可稍加润色）：

- 与**所见**的联系：别的文章、案例、现象
- 与**所想**的联系：原有观点、疑问、框架（如「司马老贼那句话」「it's hard to unlearn」）
- 与**所行**的联系：自己的实践、踩坑、工作流变化

**不追求**：全面、系统、文采。  
**追求**：即时、真实、可追溯（日后能想起「当时为什么记这一笔」）。

### 3. 整理与沉淀

- **工具**：使用「案例文章整理与优化」Prompt（或同类结构化模板）对原文做整理
- **批注的落位**：每条批注以**序号（1. 2. 3. …）**为小节标题，下挂「批注原话」+「对应原文」；批注尽量保持读者原话，仅做必要润色
- **产出**：一篇「结构化案例/长文 + 带序号的批注」文档，既是**笔记**又是**二创**，便于检索、复读、与后续输入再次勾连

---

## 🔄 良性循环

| 环节 | 作用 |
|------|------|
| **沉浸式阅读** | 有批注预期 → 读得更专注，更容易在「有火花」处停下 |
| **即时批注** | 火花被记录 → 输入与既有知识/经验显式联结，不易读过即忘 |
| **结构化整理** | 原文 + 批注一体 → 日后检索时既能看原文要点，也能看自己当时的联想与辩证 |
| **二创式沉淀** | 不是单纯摘抄，而是「原文 + 我的视角」→ 更适合作为后续写作、讨论、再阅读的磨刀石 |

---

## 📐 批注小节格式（建议）

```markdown
## 1. [简短主题/关键词]

> [批注原话，尽量保持你的原话，可稍加润色]

**对应原文**：[与该批注对应的原文摘要或关键句]
```

- 序号用 **1. 2. 3.** 即可，无须「批注一、批注二」
- 多条批注可对应同一段原文；同一主题下也可合并为一条批注

## 🛠️ 与本工作流搭配的 Prompt

- **案例/长文整理**：`案例文章整理与优化Prompt.md` —— 负责原文的结构化（概述、小节、表格、延伸阅读）
- **批注**：由读者在阅读时自行写入，整理阶段按上述格式嵌入文档，保持原话优先、序号清晰

---

## 📌 适用场景

- 想深度消化一篇长文/案例，又希望和既有知识体系挂钩
- 希望沉淀下来的不仅是「摘要」而是「我的解读 + 原文」
- 后续会反复回到同一主题，需要可检索的「当时为什么这么想」的记录
