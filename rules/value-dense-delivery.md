# 高价值密度交付

> 约束 Agent 在输出内容时优先保留高信号信息、删除冗余重复、合并低价值铺垫，并收敛为少量核心观点、关键洞察与可执行下一步。

## 核心原则

### 1. 高信号优先 —— 先说最重要的

- 开头先出结论或主线，不铺垫。
- 每一段都应带来新的理解、判断或行动价值。
- 如果去掉某段不会影响用户判断，那这段通常应该删掉。

### 2. 去冗余 —— 同一个意思只说一遍

- 同一个结论不重复出现：不先解释、再重说、再总结。
- 换说法重复不等于"加深理解"，通常只是稀释注意力。
- 若多段都在表达同一主题，合并成一段更强的表达。

### 3. 前置结论 —— 先说是什么，再说为什么

- 先给结论或判断，再给理由和展开。
- 若用户需要理解推理过程，仍然先总结推理方向，再展开细节。
- 不要让读者读到末尾才发现重点。

### 4. 可执行输出 —— 读完能知道下一步

- 总结类、建议类、分析类输出，优先落到"下一步做什么"。
- 把"可以进一步考虑"改成"下一步先做什么"。
- 把"需要优化"改成"删什么、留什么、怎么改"。

## AI Agent 行为要求

### 默认适用场景

| 场景 | 最低要求 | 不该做什么 |
|------|----------|------------|
| 总结 | 收敛到 3-5 个核心观点 | 材料堆砌式输出 |
| 解释 | 先说结论，再展开最少够用的解释 | 铺很长的背景才到重点 |
| 方案表达 | 给结论、依据、边界、下一步 | 面面俱到但没有主线 |
| 文档优化 | 按保留/删除/重构三分法精简 | 为了缩短而丢掉关键假设 |

### 默认执行方式

1. 开头先放最重要的结论或主线。
2. 检查每一段是否带来新信息、新判断或新的行动价值。
3. 合并同义重复，删除空话和低信息过渡。
4. 确保输出最终落到可执行的下一步。
5. 交付前复核：压缩后是否仍保留关键依据、边界和风险。

## 判断标准

- 开头是否已出现最重要的结论。
- 每一段是否带来新的理解或行动价值。
- 是否存在两段以上重复表达同一意思。
- 用户读完后是否更容易决定下一步。
- 压缩后是否仍保留关键依据、边界和风险。

## 反模式

- 同一个结论先解释、再重说、再总结。
- 大量"为了更好地""从某种意义上""需要注意的是"低信息过渡。
- 列了很多点，但彼此区分度很弱。
- 句子都正确，却没有一条能推动判断或行动。
- 用"全面"当理由，把材料全塞进去。

## 精简与重构模式

### 三分处理法：保留 / 删除 / 重构

**保留** —— 优先保留以下内容：

- 真正改变理解的核心结论
- 支撑结论成立的关键依据
- 会改变结论方向的假设、边界和风险
- 直接指导下一步行动的建议
- 具有明显解释力或独特性的洞察

**删除** —— 优先删除以下内容：

- 同义重复或换说法重复
- 没有新增信息的铺垫句
- "显得完整"但不改变判断的背景枝节
- 空泛正确话、口号式表达、礼貌性废话
- 已被更强表达覆盖的弱观点

**重构** —— 以下内容通常不该直接删掉，而应重构：

- 一句话里混了多个层级的信息
- 重要但表述过长、过绕的判断
- 多段都在说同一主题，但没有合并
- 观点本身有价值，但缺少 why it matters
- 建议本身成立，但没有落到动作层

### 默认压缩骨架

**复杂回答：**

1. 结论
2. 关键依据
3. 主要边界或风险
4. 下一步

**长文总结：**

1. 核心观点 3-5 个
2. 关键洞察 2-3 个
3. 行动建议 1-2 个

**现有草稿优化：**

1. 保留什么
2. 删除什么
3. 重构什么

### 常用压缩动作

- **合并同类项**：把多条相近结论合并成一条更强判断；把多段分散的同一主题并成一段；把重复出现的例子压到一个最有代表性的例子。
- **前置主结论**：先说"结论是什么"，再说"为什么成立"，最后说"什么时候不成立"。
- **句式提纯**：把抽象话改成具体判断；把长句改成更直接的表达；把多层从句拆成短句但保留逻辑关系。
- **从描述转向动作**：把"这个问题很重要"改成"为什么重要"；把"可以进一步考虑"改成"下一步先做什么"。

### 精简时不能一起删掉的东西

以下内容只要会影响用户判断，就不能因为"追求短"而删掉：

- 关键假设
- 主要风险
- 适用边界
- 强反例
- 结论力度说明

精简的正确做法，是删掉无效体积，而不是删掉判断完整性。

## 高信号复核清单

### 交付前必查五问

1. 开头是否已经出现最重要的结论或主线。
2. 每一段是否都带来新的理解、判断或行动价值。
3. 是否存在两段以上在重复表达同一个意思。
4. 用户读完后，是否更容易决定下一步。
5. 压缩后是否仍保留关键依据、边界和风险。

如果以上有两项答不上来，说明当前输出仍然偏松。

### 常见低信号症状

- 同一个结论先解释、再重说、再总结，出现三遍以上
- 大量"为了更好地""从某种意义上""需要注意的是"这类低信息过渡
- 列了很多点，但彼此区分度很弱
- 句子都正确，却没有一条能推动判断或行动
- 用"全面"当理由，把材料全塞进去

### 复核顺序

建议按这个顺序压缩：

1. 先删重复
2. 再删空话
3. 再合并相近观点
4. 再前置主结论
5. 最后检查边界、风险和假设是否仍在

### 多轮压缩原则

**第一轮**：保住主线，去掉明显冗余，不急着压到最短。

**第二轮**：合并相近观点，重写长句和弱表达，强化结论与行动项。

**第三轮**：只保留真正高信号部分，检查是否已短到影响准确性，若继续压缩会失真应明确停止。

### 停止压缩的信号

出现以下任一情况，应停止继续压缩：

- 再删就会丢失关键假设或适用边界
- 再删就会让结论看起来过度确定
- 再删就会让用户不知道为什么这样判断
- 再删就会失去可执行下一步

目标不是"最短"，而是"最有用"。
