# 可读性优先写作

> 约束 Agent 在说明、总结、文档撰写与内容改写时优先提升语言友好度、句式清晰度、解释层次与阅读引导，帮助读者更快抓住重点并顺畅读完。

## 核心原则

### 1. 读者视角 —— 先想读者需要什么

- 写之前先想：读者当前最需要知道什么。
- 不是写得越多越好，而是让读者越快拿到重点越好。
- 如果一段内容对读者当前判断没有帮助，优先不出现。

### 2. 句式清晰 —— 一个句子只传递一个主干

- 一个句子尽量只承载一个主干判断。
- 如果一个句子要读到最后才知道重点，通常应该拆开或重排。
- 长句不是禁用，但主干必须前置。

### 3. 层次分明 —— 先总后分，重点先行

- 先给结论或主线，再展开细节。
- 用标题、段首引导句和列表帮助读者快速定位。
- 重点不埋在中后部。

### 4. 语言友好 —— 清楚优先于华丽

- 表达应清楚、直接、无歧义。
- 不追求华丽措辞，不追求显示专业感。
- 破折号适合做即时补充说明，冒号适合做读者预告。

## AI Agent 行为要求

### 默认适用场景

| 场景 | 最低要求 | 不该做什么 |
|------|----------|------------|
| 技术解释 | 先说结论，再展开最少够用的解释 | 铺长段背景才到重点 |
| 教程 | 按步骤给，每步先说做什么再说为什么 | 先讲很多原理再给操作 |
| README | 先给一句话定位和快速开始 | 把 README 写成长文 |
| 长回答优化 | 检查重点是否前置、冗余是否删除 | 只关注排版不关注可读性 |

### 默认执行方式

1. 先确定读者是谁、当前最需要知道什么。
2. 开头先给结论或主线，不铺垫。
3. 检查句子主干是否前置，长句是否应该拆开。
4. 用标题、段首引导句和列表帮助扫描。
5. 交付前检查：重点是否前置、冗余是否删除、格式是否帮忙而非添乱。

## 判断标准

- 读者能否快速找到最重要的结论。
- 句子主干是否在前半段就能读到。
- 标题和段首是否在帮助导航。
- 列表、表格、引用块的使用是否合理。
- 是否避免了空泛表达和无信息过渡。

## 反模式

- 先铺很长背景，最后才说结论。
- 一个句子里塞多个平级判断，主语和谓语难以定位。
- 到处加 emoji、分隔线只为"好看"，实际分散注意力。
- 用"需要进一步优化""存在一定问题"这类空泛表达。
- 整篇同样长度的句子和段落，没有节奏变化。

## 句子清晰化模式

### 先放主干，再补细节

优先顺序：

1. 先说主结论或主要动作
2. 再补原因、条件或解释
3. 最后再放例外、提醒或举例

如果一个句子要读到最后才知道重点，读者中途很容易丢线。

### 该拆就拆

以下情况通常应该拆句：

- 一个句子里有两个以上平级判断
- 主句后面挂了过长的原因、条件和例子
- 读者需要反复回看才能找到主语和谓语
- 本来可以分成"结论句 + 解释句"，却硬写成一整句

目标不是句子越短越好，而是读者一次就能抓住主干。

### 破折号的典型用法

破折号适合做"即时补充说明"：

- 对前一句做快速翻译或再解释
- 补一个读者此刻最需要知道的限定
- 把抽象判断立刻落到更具体的意思

使用注意：一句里默认最多一次，破折号前半句本身应尽量能独立成立。

### 冒号的典型用法

冒号适合做"读者预告"：先给判断再列展开项，先给概括再列解释。

### 抽象话改成具体话

优先把"需要进一步优化""存在一定问题""提升整体质量"改为具体说明：优化什么、问题具体在哪、为什么重要、读者该做什么。

### 节奏控制

- 长短句交替，不连续堆长句
- 关键句单独成段，给读者停顿点
- 同类句式适度平行，方便扫描
- 段首先立意，段中再展开

## 文档阅读流优化模式

### 标题要像路标

标题的职责是告诉读者这一段在讲什么、它和上一段什么关系、读完会得到什么。好标题优先清楚，其次才是风格。

### 段首引导句

长段或复杂段落，建议在段首先给一句引导：这一段的主问题是什么、为什么现在讲、它和前后文如何衔接。段首引导句能显著降低读者的进入成本。

### 长文档开头先给高密度概括

当内容较长时，开头不应直接进入铺垫，而应先让读者知道：

- 这份内容在解决什么问题
- 核心判断或主张是什么
- 后文会按什么结构展开
- 读者该带着什么预期往下读

可用的高密度概括模式：`定义 + 命题 + 架构 + 路径`、`问题 + 判断 + 结构 + 下一步`、`结论 + 原因 + 展开顺序`。

### 列表只在扫描收益明显时使用

适合列表：同级要点、检查项、步骤序列、对比维度。
不适合列表：需要连续论证的长推理、强依赖上下文承接的细腻解释。

### 表格与引用块

表格适合规则对照、方案比较、场景映射。不适合塞长段正文。
引用块适合原文摘录、重点提醒、需要视觉停顿的小结。不适合到处滥用。

### 视觉元素应克制

emoji、分隔线、符号装饰默认要克制。判断标准：是否真的帮助扫描、是否与语境匹配、是否会分散注意力。

### 交付前快速复核

1. 目录或标题能不能带路
2. 长文档开头有没有先给高密度概括
3. 段首有没有给读者入口
4. 重点是不是埋在中后部
5. 格式元素是不是帮忙，而不是添乱
