# 提交前检查清单

## 1. 结构检查

- [ ] H1 仅用于文档主标题，无编号
- [ ] H2/H3/H4 均有编号且连续
- [ ] 文档长度 ≤ 300 行
- [ ] 无空标题
- [ ] 无孤立列表项

## 2. 格式检查

- [ ] 所有路径使用反引号包裹
- [ ] 代码块注明语言类型
- [ ] 强制要求使用 `**⚠️ 强制要求**` 格式
- [ ] 无未闭合代码块

## 3. 日期检查

- [ ] 使用当前系统日期 `YYYY-MM-DD`
- [ ] 月份和日期为 2 位数字
- [ ] 文件名日期与内容一致
- [ ] 无占位符未替换

## 4. 内容检查

- [ ] 标题准确描述章节内容
- [ ] 无重复内容
- [ ] 引用路径正确
- [ ] 无拼写错误

## 5. 快速修复参考

| 问题 | 修复方法 |
|------|----------|
| 无编号的 H2/H3/H4 | 立即补齐编号 |
| 日期格式错误 | 使用 `date +%Y-%m-%d` 获取正确格式 |
| 文档超 300 行 | 拆分并在 README 记录原因 |
| 路径未加反引号 | 添加反引号 |
| 代码块无语言标注 | 添加语言类型 |

## 6. 提交命令示例

```bash
# 检查日期格式
date +%Y-%m-%d

# 检查文件行数
wc -l document.md

# 检查 Markdown 语法（如有 markdownlint）
markdownlint document.md
```
