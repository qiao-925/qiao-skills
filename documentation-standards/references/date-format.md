# 日期格式详细说明

## 1. 标准格式

所有日期使用 `YYYY-MM-DD` 格式：

| 元素 | 格式 | 示例 |
|------|------|------|
| 年份 | 4位数字 | `2026` |
| 月份 | 2位数字（补零）| `01`, `12` |
| 日期 | 2位数字（补零）| `01`, `31` |

## 2. 获取当前日期

```bash
# Linux/macOS
date +%Y-%m-%d

# 输出示例
2026-01-24
```

## 3. 文件名日期

文件名含日期时，格式为：

```
YYYY-MM-DD-N_任务名称_类型.md
```

示例：
- `2026-01-24-1_Skills拆分_实施计划.md`
- `2026-01-24-2_Bug修复_完成总结.md`

## 4. 文档内日期

### 4.1 版本信息

```markdown
## 版本信息

- **最后更新**：2026-01-24
- **版本**：v1.0
```

### 4.2 状态表

```markdown
| CP | 状态 | 完成日期 |
|----|------|----------|
| CP1 | ✅ | 2026-01-24 |
| CP2 | ⬜ | - |
```

## 5. 自查清单

更新日期后检查：

- [ ] 占位符是否已替换
- [ ] 月份是否为 2 位数字（01-12）
- [ ] 日期是否为 2 位数字（01-31）
- [ ] 文件名日期是否与内容一致

## 6. 常见错误

| 错误 | 正确 |
|------|------|
| `2026-1-24` | `2026-01-24` |
| `24-01-2026` | `2026-01-24` |
| `2026/01/24` | `2026-01-24` |
| `Jan 24, 2026` | `2026-01-24` |
