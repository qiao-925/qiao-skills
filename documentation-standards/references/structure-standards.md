# 结构与标题详细规范

## 1. 标题层级规范

### 1.1 H1 使用规则

- H1 仅用于文档主标题
- 每个文档只能有一个 H1
- H1 禁止编号

```markdown
# 文档主标题

## 1. 第一章
### 1.1 第一节
```

### 1.2 编号规则

| 层级 | 格式 | 示例 |
|------|------|------|
| H2 | `## N. 标题` | `## 1. 概述` |
| H3 | `### N.M 标题` | `### 1.1 背景` |
| H4 | `#### N.M.K 标题` | `#### 1.1.1 问题` |

### 1.3 编号连续性

- 同级标题编号必须连续
- 新增或删除章节时同步调整编号

```markdown
# 正确示例
## 1. 第一章
## 2. 第二章
## 3. 第三章

# 错误示例
## 1. 第一章
## 3. 第三章  # 缺少第二章
```

## 2. 篇幅管理

### 2.1 长度限制

| 文档类型 | 建议长度 | 最大长度 |
|----------|----------|----------|
| 普通文档 | ≤ 200 行 | 300 行 |
| 扩展文档 | ≤ 400 行 | 500 行（需用户确认）|

### 2.2 拆分原则

- 按主题/职责拆分
- 避免重复内容
- 拆分后在 README 记录拆分原因

### 2.3 引用处理

- 大段信息改为链接到参考文档
- 正文只保留关键结论

```markdown
# 正确：引用参考
详细说明请参考 `references/detailed-guide.md`。

# 错误：大段复制
（大段内容直接复制到正文）
```

## 3. 格式规范

### 3.1 引用路径

所有路径使用反引号包裹：

```markdown
# 正确
请参考 `agent-task-log/` 目录。

# 错误
请参考 agent-task-log/ 目录。
```

### 3.2 代码块

注明语言类型：

```markdown
# 正确
` ` `bash
echo "hello"
` ` `

# 错误
` ` `
echo "hello"
` ` `
```

### 3.3 强调格式

- 强制要求：`**⚠️ 强制要求**`
- 警告：`⚠️ 注意`
- 禁止：`❌ 禁止`
- 推荐：`✅ 推荐`
