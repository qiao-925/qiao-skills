# 文档编写规范

> 约束项目内 Markdown / 规则 / 日志等文档的写作方式，确保结构清晰、信息一致。

---

## ⚠️ 核心强制要求

### 结构与标题

- H1 仅用于文档主标题，禁止编号
- H2 采用 `## 1. 标题`、H3 `### 1.1 标题` 格式
- 同级标题编号保持连续

### 篇幅管理

- 单个文档长度 ≤ 300 行
- 引用大段信息改为链接到参考文档

### 日期格式

- 所有日期使用 `YYYY-MM-DD` 格式
- 文件名日期与内容一致

---

## AI Agent 行为要求

### 创建/修改文档时

- 确保标题编号连续
- 检查日期格式正确
- 控制篇幅在 300 行内

### 提交前检查

- [ ] 所有路径使用反引号包裹
- [ ] 代码块注明语言类型
- [ ] 无空标题、无孤立列表项
