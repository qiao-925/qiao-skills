# 架构变更影响面分析模板

## 1. 变更定位

- 变更对象：
- 所在层级：
- 主要职责：

---

## 2. 影响分析

### 2.1 上游调用方

- 谁在调用该模块/接口？
- 是否需要修改调用约定？

### 2.2 下游依赖

- 该模块依赖哪些能力？
- 是否引入跨层或反向依赖风险？

### 2.3 数据流

- 输入数据来源是否变化？
- 输出格式/语义是否变化？

---

## 3. 风险分级

### 低风险

- 仅内部重构，不改外部接口

### 中风险

- 扩展接口（向后兼容）

### 高风险

- 破坏性接口变更
- 数据流主路径调整
- 分层结构或依赖方向变化

---

## 4. 结论与动作

- 是否需要用户决策：是 / 否
- 兼容策略：默认值 / 适配层 / 版本化
- 验证动作：依赖检查 / 关键路径测试 / 回归测试
